\section{Examples for the Export of Contributions}

The following sections contain content (about an imaginary topic) based on which an RDF document is constructed by RDFtex's preprocessor. The document is serialized and stored at \texttt{./exports.ttl}.

\subsection{Definition Export}

Here, we introduce the term \emph{noipper}. \rdfproperty{Noipper Definition}{mskg:definition_content}{A noipper is the main technical component of a noipping machine. Its purpose is to nullify the interference that is caused by exceedingly high neipping values in the system.}

\subsection{Dataset Export}

Of course, there is a multitude of available noippers from different brands. \rdfproperty{noipperbase}{mskg:dataset_description}{We compiled an exhaustive dataset that lists all available noippers and their features to allow for a fair comparison}. The dataset is called \rdfproperty{noipperbase}{mskg:dataset_name}{noipperbase} and is available at \rdfproperty{noipperbase}{mskg:dataset_url}{https://example.org/datasets/noipperbase}.

\subsection{Experimental Result Export}

To ensure safe operation, the noipper has to nullify the interference as fast as possible. \rdfproperty{Neipping Duration}{mskg:expresult_description}{We tested a readily available noipper in a typical neipping system and monitored the time it takes to nullify the interference when exceedingly high neipping values are encountered.} On average, the process took \rdfproperty{Neipping Duration}{mskg:expresult_result}{0.123 seconds} based on \rdfproperty{Neipping Duration}{mskg:expresult_samplesize}{100000000} runs.
